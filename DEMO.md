# MD Reviewer 試用文件

歡迎試用！這份是讓你體驗「**選字 → 加註 → Claude 讀回**」迴路的文件。

## 怎麼玩

1. 用滑鼠**選取**下面任何一段文字。
2. 跳出小框 → 選顏色、寫意見 → 按「加註」（或 `Ctrl+Enter`）。
3. 右邊欄出現你的註解、左邊文字被高亮；改錯了可在卡片上「刪除」或「標記解決」。
4. 加完跟我說「看一下 DEMO.md 的審閱」,我就讀回你的紅筆續做。

## 幾句故意有問題的話,挑來改改看

- 這個系統的吞吐量上限要用壓測數據佐證;「每秒一億筆」目前沒有依據(無機器規格、無 benchmark),先別寫死。另外高吞吐通常**更**需要快取擋重複查詢,而非「不需要」。
- 密碼直接存明文進資料庫最方便,反正內網很安全。
- 這段邏輯一定不會出錯,所以不用寫測試。

## 渲染能力一覽

支援 **粗體**、*斜體*、`行內碼`、~~刪除線~~、[連結](https://example.com)。

| 元素 | 支援 | 備註 |
|------|------|------|
| 表格 | ✅ | 你現在看到的 |
| 巢狀清單 | ✅ | 見下方 |
| 圍欄程式碼 | ✅ | 見下方 |

- 巢狀清單
  - 第二層
    - 第三層
- 待辦清單
  - [x] 已完成的項目
  - [ ] 還沒做的項目

```js
function review(doc) {
  // 選這段程式碼也能加註
  return doc.annotations.filter(a => a.status === "open");
}
```

> 引用區塊裡的文字也能精準標注 —— 選這一句試試看。

## Mermaid 圖表

支援 ` ```mermaid ` 圖表（內建離線渲染,不對外連線）:

```mermaid
flowchart LR
  A[Claude 產出 .md] --> B[你選字加註]
  B --> C[存 *.review.json]
  C --> D[Claude 讀回續做]
  D --> A
```
